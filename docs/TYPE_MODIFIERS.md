* short - экономия памяти, 16 бит
* long / long long — когда int уже не хватает
* unsigned — две главные причины - 1. `unsigned` - Первое! В 2 раза больше положительный диапазон. unsigned int → 0 … 4 млрд вместо –2 млрд, примерно. Очень часто отрицательные значения просто не имеют смысла : { размер массива, количество байт, цвет (RGB 0–255), пиксельные координаты, битовые флаги } Воопщем! unsigned спасает от случайного переполнения в отрицательную сторону И второе! 
* signed (по умолчанию) — потому что большинству людей удобнее думать отрицательными числами
